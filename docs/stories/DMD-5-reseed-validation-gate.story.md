## Story

**As the** Figmento maintainer,
**I want** every seeded design system in `knowledge/design-systems/` to ship with a canonical `DESIGN.md` file generated by the official exporter,
**so that** (a) the repo's DESIGN.md set becomes a regression baseline for future schema changes, (b) other tools (Cursor, Claude Desktop, Cline) that consume Figmento's knowledge directory can read the markdown directly, and (c) Phase A closes with verifiable end-to-end coverage across all 7 systems.

## Background

DMD-1 committed 3 reference samples (notion, aurelia, claude). DMD-4's round-trip test suite proved that `figma`, `linear`, `stripe`, `vercel` can be exported, but no file was ever written to disk — the tests operate on in-memory strings.

DMD-5 closes that loop: run the official exporter against each tokens.yaml that lacks a DESIGN.md, write the markdown next to the yaml, and commit both as the canonical pair. If the exporter ever regresses, regenerating the file will produce a visible git diff — turning the committed DESIGN.md set into a regression baseline.

## Acceptance Criteria

1. **AC1 — Seed script exists.** A runnable script at `figmento-mcp-server/scripts/seed-design-md.js` drives the export. Supports `--check` (dry-run) and `--force` (overwrite existing DESIGN.md files).
2. **AC2 — All 7 systems have DESIGN.md.** After the seed runs, every directory under `knowledge/design-systems/` contains both `tokens.yaml` and `DESIGN.md`.
3. **AC3 — Round-trip PASS for all generated files.** Each newly written DESIGN.md validates with verdict `PASS` under `validateDesignMdIR`.
4. **AC4 — No regression in existing samples.** notion/aurelia/claude DESIGN.md files are skipped by default (only `--force` re-writes them); existing round-trip tests continue to pass.
5. **AC5 — Full test suite green.** `npx jest --testPathPattern='ds-md'` reports 86/86 PASS after seeding.
6. **AC6 — Script produces deterministic output.** Running `--check` twice on a clean tree produces identical results; running without `--check` on a clean tree produces the same bytes as the last committed DESIGN.md.
