## Description

Allow designers to export their current plugin configuration — format preferences, style/theme preferences, brand kit, and design system summary — as a Figma Skill markdown file that developers can use with Claude Code + the `use_figma` MCP tool. This bridges the designer-developer gap: a designer configures the plugin once, exports a `.md` skill file, and any developer on the team can produce designs that respect the designer's system.

### What Gets Exported

The skill file is a self-contained markdown document (following the format established by FN-1's `skills/figmento-screenshot-to-layout.md`) that encodes:

1. **Design System Summary** — From `designSystemState.cache` (FN-6): component names by category, variable names with resolved values (colors, spacing), paint/text/effect style names. This gives the developer's Claude Code agent awareness of available components and tokens.

2. **Brand Kit** — From `designSettings` in `state.ts`: selected font family, brand colors (hex values), custom prompt instructions. If a brand kit has been saved via `save_brand_kit` (CF-3), include the full brand identity (primary/secondary/accent colors, fonts, voice).

3. **Format Preferences** — From `modeState` in `state.ts`: preferred social format, color theme, reference images description. From chat settings: preferred model, generation style.

4. **Learned Preferences** — From `clientStorage` key `figmento-preferences` (LC-11): the aggregated preference rules (e.g., "Always use Playfair Display for headings", "Prefer dark backgrounds", "CTA buttons should be rounded"). These are the distilled lessons from the designer's correction history.

5. **Usage Instructions** — How to load the skill in Claude Code, what MCP tools to use (`use_figma`), and workflow guidance adapted from the existing skill format.

### Skill File Format

The exported file follows the Figma Skill markdown convention established in FN-1:

```markdown
# [Brand Name] Design System — Figma Skill

## Prerequisites
- Claude Code with `use_figma` MCP tool connected

## Brand Identity
[Colors, fonts, voice from brand kit]

## Design System
### Components
[Categorized component list with names]
### Variables
[Variable names → resolved values, grouped by collection]
### Styles
[Paint styles, text styles, effect styles]

## Design Preferences
[Learned preferences as design rules]

## Format Defaults
[Preferred format, color theme, spacing preferences]

## Workflow
[How to use this skill when designing]
```

### "Share My Workflow as a Skill"

The export UI is a button in the Settings sheet or Status Tab (FN-15) labeled "Share My Workflow as a Skill". Clicking it:
1. Gathers all exportable data from state and `clientStorage`
2. Generates the markdown content
3. Triggers a download of the `.md` file (via `Blob` + `URL.createObjectURL` + click)

The filename is derived from the brand name or Figma file name: `[brand-name]-design-skill.md`.

### Why This Is Novel

No other Figma tool exports designer configuration as a portable skill file. This creates a new workflow: designer configures once in Figma, developer uses the configuration in Claude Code. The skill file is human-readable (markdown), version-controllable (git), and shareable (paste in Slack, commit to repo).

## Technical Notes

### File Download in Figma Plugin iframe

Figma plugin iframes use `data:` URLs, which restrict some browser APIs. The file download pattern that works:

```typescript
function downloadFile(content: string, filename: string) {
  const blob = new Blob([content], { type: 'text/markdown' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
}
```

**Caveat:** `URL.createObjectURL` may not work in all Figma plugin iframe contexts due to `data:` origin restrictions. Fallback: encode as base64 data URL and open in new tab for manual save. Test during implementation.

**Alternative approach:** Send the markdown content to the sandbox via `postMessage({ type: 'export-skill', content })`, have the sandbox call `figma.ui.postMessage({ type: 'download-skill', content })` back (this round-trip is needed if direct download fails from the iframe).

### Data Sources

| Data | Source | Access Pattern |
|------|--------|---------------|
| Components, variables, styles | `designSystemState.cache` | Direct read from `state.ts` |
| Brand colors, font, custom prompt | `designSettings` | Direct read from `state.ts` |
| Brand kit (CF-3) | `clientStorage: figmento-brand-kit` | `postMessage({ type: 'get-brand-kit' })` → async response |
| Learned preferences (LC-11) | `clientStorage: figmento-preferences` | `postMessage({ type: 'get-preferences' })` → async response |
| Format, color theme | `modeState` | Direct read from `state.ts` |

Since brand kit and preferences require async `clientStorage` reads, the export flow is:
1. User clicks "Share My Workflow as a Skill"
2. UI sends `get-brand-kit` and `get-preferences` messages to sandbox
3. UI waits for both responses
4. UI generates markdown from all gathered data
5. UI triggers download

### Skill Format Reference

The FN-1 skill (`skills/figmento-screenshot-to-layout.md`) establishes the format:
- Title as H1
- Prerequisites section
- Workflow section with numbered steps
- Knowledge tables (formats, fonts, colors, etc.)
- Code examples for `use_figma`
- Self-evaluation checklist

The exported skill follows the same structure but replaces generic knowledge with the designer's specific system: their components instead of size presets, their variables instead of generic palettes, their preferences instead of default rules.

### Variable Value Extraction

`designSystemState.cache.variables` contains objects with `{ id, name, resolvedType, valuesByMode }`. For the skill export, resolve the default mode value:
- `resolvedType: 'COLOR'` → extract as hex (using `rgbToHex` from `color-utils.ts`)
- `resolvedType: 'FLOAT'` → extract as number
- `resolvedType: 'STRING'` → extract as string

Group variables by collection name for readability.
